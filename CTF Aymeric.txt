prob controlle acces

1 connecter vous a l'adresse du serveur http://Adresse:8080


2 Envoyer une requête Post signup d'on le raw body en JSON contient un compte avec le Nom de votre choisi et le mot de passe de 
votre choix http://Adresse:8080/api/id/signup pour crée un compte

{
    "username" : "NOM",
    "password" : "Mot de passe"
}


3 Envoyer une requête Post add pour crée une tache assurez vous que
le raw body contient le nom de la tache. http://Adresse:8080/api/add

{

    "name" : "test7"

}

4 Envoyer une requête Get pour voir la liste de tache du compte que vous avez créé cette
requete n'a pas de body  http://Adresse:8080/api/home

5 Dans le response body rechercher la tache que vous avez crée elle devrais ressembler a ceci
{
        "id": 5,
        "name": "Nom",
        "percentageDone": 0,
        "percentageTimeSpent": 0.0,
        "deadline": "2025-05-08T13:41:29"
    } 
	
	Récupéré le idMAX de cette tache il représente la taille total de la liste

6 Faite un boucle qui part de 1 jusqu'à l'idMAX récupéré qui
envoi une requête GET sans body http://Adresse:8080/api/detail/{id} ou on ne fait que changer le id
 jusquà ce qu'on trouve la tache désiré. Pour trouver la tâche désiré, vérifier le name de la tache dans le response body. 
 {
    "id": 1,
    "name": "test",
    "deadline": "2025-05-08T13:18:03",
    "events": [
        {
            "value": 69,
            "timestamp": "2025-05-01T13:21:24"
        }
    ],
    "percentageDone": 69,
    "percentageTimeSpent": 0.0
}
 
 

7 Une fois la tache désiré trouvé. Garder en tête l'id qui a été mis dans la requête http://Adresse:8080/api/detail/{ID}
Et Envoyer une requête GET sans body qui contient l'ID et 
la valeur que le souhaite changer. http://Adresse:8080/api/progress/{ID}/{VALEUR}


8 Envoyer une requête GET pour vérifier que la valeur a bien été modifier http://Adresse:8080/api/detail/{ID}
